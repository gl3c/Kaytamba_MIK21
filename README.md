# DescScore AI
Веб-приложение для автоматической оценки качества описаний товаров с использованием ИИ и эвристических правил.

Проект позволяет оценить описание товара по нескольким критериям, выставить балл от 0 до 10 и получить рекомендации по улучшению.  
Система ориентирована на продавцов маркетплейсов (Wildberries, Ozon, Avito, Яндекс.Маркет).

---

##  Основные возможности

- Оценка качества описания **от 0 до 10**
- Вердикт: «Отличное», «Хорошее», «Среднее», «Слабое»
- Рекомендации по улучшению
- Генерация улучшенного варианта описания
- Красивый и удобный веб-интерфейс
- Простая и быстрая работа (без тяжёлых моделей)
---

##  Используемые технологии

- **Python**
- **Flask** (веб-сервер)
- **HTML + Tailwind CSS + JavaScript**
- Эвристический анализ (длина, разнообразие слов, структура текста)
- Jinja2 (шаблоны)
---

##  Архитектура проекта

Проект построен по модульному принципу:

- `app.py` — запуск веб-сервера и обработка запросов
- `templates/index.html` — пользовательский интерфейс
- `evaluate.py` — основная логика оценки (эвристики)
- `train.py` — обучение модели (если используется)
- `data.py` — подготовка данных
- `models/` — папка для сохранения обученной модели

## 🚀 Установка и запуск (для пользователя)

1. Клонировать репозиторий:

```bash
git clone https://github.com/твой-логин/descscore-ai.git
cd descscore-ai
```

2. Создать виртуальное окружение (рекомендуется):
```bash
Bashpython -m venv .venv
```
3. Активировать окружение:
```bash
Windows:
Bash.venv\Scripts\activate
Linux / macOS:
Bashsource .venv/bin/activate
```
4. Установить зависимости:
```bash
Bashpython -m pip install -r requirements.txt
```
5. Запустить приложение:
```bash
Bashpython app.py
```

3. Формирование вердикта и рекомендаций
4. Генерация улучшенного описания

##  Отладка

Сервер не запускается → проверьте порт 5000
Ошибка при оценке → описание слишком короткое (минимум 20 символов)
Модель не найдена → при первом запуске app.py автоматически запускает обучение

## Реализация проекта в соответствии с методологией CRISP-DM
## 1. Постановка задачи (Business Understanding)

Цель проекта — разработка удобного инструмента, который помогает продавцам на маркетплейсах (Wildberries, Ozon, Avito, Яндекс.Маркет) быстро оценивать качество описаний товаров и улучшать их.
Проблема заключается в том, что большинство описаний товаров пишутся некачественно: они слишком короткие, не содержат ключевых характеристик, преимуществ и SEO-слов, из-за чего снижается конверсия и позиции в поиске.

Задача проекта — создать автоматическую систему, которая оценивает описание по шкале от 0 до 10, выдаёт понятный вердикт и предлагает рекомендации по улучшению.

## 2. Анализ данных (Data Understanding)

Для проекта были собраны реальные описания товаров с маркетплейсов, а также созданы синтетические примеры разного уровня качества.

Анализ показал, что качественные описания отличаются большей длиной текста, высоким разнообразием словаря, наличием конкретных технических характеристик и акцентом на преимуществах для покупателя.
Были выявлены основные факторы, влияющие на качество: объём текста, лексическое разнообразие и структурированность информации.

## 3. Подготовка данных (Data Preparation)

Данные были приведены к единому формату. Для обучения и тестирования создана синтетическая выборка из 1000+ описаний товаров с присвоенными метками качества от 1.0 до 10.0.
Текст был очищен от лишних символов, приведён к нижнему регистру и разбит на признаки (длина, количество уникальных слов, наличие ключевых конструкций).

Таким образом, был сформирован готовый датасет для дальнейшего моделирования.

## 4. Моделирование (Modeling)

В проекте реализован гибридный подход: основная оценка качества построена на эвристических правилах (длина описания, разнообразие лексики, наличие характеристик).
При необходимости предусмотрена возможность дообучения небольшой нейронной модели (ruBERT-tiny).

Такое решение позволило добиться высокой скорости работы и хорошей точности без использования тяжёлых вычислительных ресурсов.


## 5. Оценка результатов (Evaluation)

Проект успешно решает поставленную задачу: система быстро и точно оценивает описания товаров, выдаёт понятный вердикт и генерирует улучшенный вариант текста.
Тестирование показало, что инструмент полезен как для начинающих, так и для опытных продавцов.

Время отклика — менее 1 секунды, что делает его удобным для реального использования на маркетплейсах.


## 6. Внедрение (Deployment)

Проект реализован в виде веб-приложения на базе Flask с современным HTML-интерфейсом.
Запуск осуществляется одной командой python app.py. Приложение работает локально на любом компьютере с установленным Python и не требует мощного оборудования.

Интерфейс интуитивно понятен, не требует регистрации и установки дополнительного ПО. В дальнейшем проект может быть легко развёрнут на облачных платформах (Render, PythonAnywhere, Heroku, VPS), превратившись в полноценный онлайн-сервис для продавцов маркетплейсов.
